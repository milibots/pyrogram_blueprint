🚀 Pyrogram Bot Blueprint Generator

GitHub License Python

Stop rewriting the same boilerplate code for every Telegram bot! This script
instantly generates a fully functional, production-ready Pyrogram + SQLAlchemy
bot architecture in seconds.

✨ What's Inside?

A carefully crafted Python project structure organized for scaling and clean
coding:

📦 Smart Architecture

  - main.py — Clean entry point with automatic database initialization
  - plugins/ — Smart Pyrogram routing (auto-loads commands & callbacks)
  - helpers/ — Utility modules, DB connections, and decorators

🗄️ Database & Models

  - Built-in SQLAlchemy + SQLite integration (bot.db)
  - Auto-migration & table creation on startup
  - Pre-configured User model

🔐 Configuration Management

  - .env — Secure storage for BOT_TOKEN, API_ID, and API_HASH
  - settings.json — Live-updatable settings (Admins, Support IDs, etc.)
  - config.py & texts.py — Object-oriented config wrappers

⚙️ Magic Decorators

  - @ensure_user — Automatically registers new users into the database before
    executing command handlers!

🚀 Instant Installation

One command to rule them all: Create your new bot project in seconds!

bash -c "$(curl -fsSL https://raw.githubusercontent.com/milibots/pyrogram_blueprint/main/blueprint.sh)"

Note: Replace milibots with your actual GitHub username if hosted elsewhere.

That's it! The script will: ✅ Ask you for your project name ✅ Create the
standard bot directory structure ✅ Generate all foundational Python files ✅
Prepare the requirements.txt ready for installation

🎯 Quick Start Guide

Once the script finishes, just run these 4 simple steps:

# 1. Enter your new project directory
cd your_project_name

# 2. Add your Telegram credentials to the .env file
nano .env 

# 3. Install dependencies
pip install -r requirements.txt

# 4. Start your bot!
python main.py

📋 Complete File Structure

Here is the exact structure generated by the blueprint:

| File / Folder     | Purpose                                                   |
| :---------------- | :-------------------------------------------------------- |
| 📄 `main.py`       | Starts the Pyrogram client and initializes DB tables.     |
| 📄 `config.py`     | Handles all `.env` and `settings.json` variables.         |
| 📄 `texts.py`      | Wrapper for `texts.json` to keep your bot messages clean. |
| 📄 `.env`          | Stores `API_ID`, `API_HASH`, and `BOT_TOKEN`.             |
| 📄 `settings.json` | Live configurations (Admins list, dynamic settings).      |
| 📄 `texts.json`    | Stores your bot's text responses.                         |
| 📁 `helpers/`      | Directory for all background logic and database files.    |
| ↳ `db.py`         | SQLAlchemy engine and session makers.                     |
| ↳ `models.py`     | Database schemas (Includes the base `User` table).        |
| ↳ `decorators.py` | Includes `@ensure_user` to auto-save users to the DB.     |
| ↳ `kb.py`         | Centralized InlineKeyboard generator class.               |
| 📁 `plugins/`      | Pyrogram automatically loads handlers from here.          |
| ↳ `commands.py`   | Standard message handlers (e.g., `/start`).               |
| ↳ `inline.py`     | Callback query handlers for your inline keyboards.        |

🔧 Manual Installation

Prefer to check the script first before running it directly?

# Download and inspect
curl -fsSL https://raw.githubusercontent.com/milibots/pyrogram_blueprint/main/blueprint.sh -o blueprint.sh

# Review the script
cat blueprint.sh

# Make it executable and run locally
chmod +x blueprint.sh
./blueprint.sh

🗑️ Uninstallation / Cleanup

Because this is a project generator, it doesn't install anything globally on
your system. To remove a generated bot, simply delete the folder:

rm -rf your_project_name
